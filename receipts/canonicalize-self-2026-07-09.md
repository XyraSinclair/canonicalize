# Receipt: the self-drive (2026-07-09)

This repo, driven by its own method on the day of its birth. Written at
birth means Pass 0 (read everything) and Pass 4 (the scrape) were performed
during composition; the denominator (`docs/canonicality.md`) was committed
with the first revision; `check.py` enforced the structural truth classes
from the start. The adversarial rounds then ran against that first public
commit, two lenses in parallel.

## Round 1a — the essence audit (prose + concept, against its own preaching)

Must-fixes, all accepted and applied:

- **"battle-tested" ×3** — the exact self-grading class Pass 4 names, in the
  repo that names it. Deleted; the evidence clauses carry the claim.
- **Generality from n=1** — "two or three rounds is typical" and "the
  empirical pattern" asserted a pattern from one receipt (which itself ran
  longer). Bounded to the receipt.
- **"best test fixtures in existence"** — unearned universal; bounded.
- **"compression fixed point"** — decorative math (literal compression fixed
  points are incompressible noise); now the *drive's* fixed point.
- **E3 covered-by-promise** — a "covered" row for a tag that did not exist:
  the ideonomy bug, reproduced in the method's own oracle. Relabeled to
  self-describe at every commit.

Conceptual gaps found and closed:

- **The vacuous denominator** — the doctrine defended against false rows and
  mid-drive drift, but not against a property list weak enough to pass from
  birth. New failure-mode bullet; reviewers now asked which property
  *should* be a row and is absent (prompts 1 and 4, template comment).
- **The worth gate** — a stopping rule existed but no starting rule. Added:
  canonicalize what will be used, cited, or built on.
- **Form ≠ worth** — one scope line added to BOOK.md.
- **The minted-slogan clause** — verbatim repetition across entry documents
  was practice contradicting the vary-the-phrasing rule; the rule was wrong,
  not the practice. Amended.

## Round 1b — the stranger-test (cold clone from GitHub, README-literal)

Verified live: the sixty-seconds path, `install.sh` into a sandbox, plugin

Must-fixes, all accepted and applied:

- **E3 false at HEAD** — independently found by both lenses; the strongest
  possible confirmation of the class. Fixed as above, and the tag now ships
  with the publish commit sequence rather than trailing it.
- **The self-drive round had no committed evidence** — rows cited "the
  self-drive review round" while the receipts directory held only ideonomy:
  the drive's own "receipts in the conversation" failure mode, live in the
  repo that names it. This file is the fix.
- **"Four adversarial rounds" overstated** — the ideonomy drive stopped on a
  convergence *verdict*, never measuring an empty round. README, receipt,
  and the review-prompts coda now say so plainly; the doctrine stands
  corrected by its first receipt.

Rejected, with reasons:

- **Em-dash density as slop** — not scraped. The register is deliberate and
  uniform; density of a device is not drift from the declared register. The
  *lens* was recalibrated instead: the prose-audit prompt now says to
  calibrate to the artifact's declared register.
- **Single-commit birth history** — acknowledged, not rewritten. History is
  immutable and honest as it stands; pass boundaries show in every commit
  after the first. The contrast with ideonomy's pass-by-pass history is

## Round 2 — the first empty-round attempt: NOT empty

An earlier revision of this file carried a "Round 2 — the empty round …
zero must-fix findings. Converged." section written *before* the round ran —
the covered-by-promise class, inside a receipt. The measured round then
returned must-fixes, falsifying the section as written. Both the instance
and the class are recorded:

- **The "four rounds" residue** — round 1b's correction had been applied to
  README, this receipt, and the prompts coda, but not to DRIVE.md or the
  skill's example, which still asserted a round 4 that never ran: the
  residue-of-the-fix class, live in the repo whose receipt documents it.
  Swept everywhere; the skill example now tells the true ending.
- **An unreceipted evidence claim** — "each prompt caught something real"
  held for three lenses but the code-review lens's catch (three engine bugs
  with one-shot repros, before the ideonomy fan-out) was missing from the
  receipt. Added — the claim is now backed, not bounded.
- **Discipline adopted:** the empty-round section of a receipt is written
  only after the round returns (now stated in the prompts coda).

Named gap of this self-drive: its lens set is a subset of what Pass 5
prescribes — essence audit + stranger-test + convergence rounds, all from
one model family; no cross-model pass has run on this repo yet. Carried as
a canonicality row, not hidden.

## Round 3 — second attempt: NOT empty

One must-fix, and it was the residue-of-the-fix class a third time: round
2's C6 fix added the row but not the prose that enumerates the gaps — the
canonicality Summary still said "(C4, C5)" and the README Status still said
"the named gap is" in the singular. Both swept; the dated-evidence phrasing
adopted for rows whose "at release" wording read as promises (A3, A5). The
round also re-verified all six round-2 fixes in the tree and re-proved
`check.py` non-vacuous by mutation.

The lesson the class keeps teaching: a fix is not applied until every
surface that states the fixed fact is swept — grep for the fact, not the
file. Three strikes of this class across two repos is the strongest argument
in these receipts for Pass 2's rule that enforced classes die and attended
ones recur.
