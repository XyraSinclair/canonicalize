# Receipt: canonicalizing XyraSinclair/ideonomy (2026-07-09)

The drive this repo distills, as it actually ran. Target:
[github.com/XyraSinclair/ideonomy](https://github.com/XyraSinclair/ideonomy)
— already public, substantive, and twice externally reviewed before the
drive began; "already good" is where canonicalization starts, not where it
stops. One session, passes 0–6, three adversarial rounds plus a convergence
verdict, released as v0.2.0. (Defect tallies below are the session's triage
counts; the rounds, fixes, tags, tests, and residue are reconstructible from
the target's public history — the tallies are not.)

## What the full read found (Pass 0) that two prior reviews had not

- The flagship doc said **35 primitives**; the code shipped 37; the demo
  printed "35" in a run that self-reported 37 groups two lines later.
- Links to two sibling repos that had **never been published**.
- The docs claimed only "the top three" skills were scaffolded; all 14 were.
- Python version contradicted between packaging and docs (and both were
  wrong — the suite passed on an older runtime than either claimed).
- A private strategy document referenced by path in a public file.

## What enforcement fixed as a class (Pass 2)

`test_consistency.py`: doc↔code catalog sync both directions, prose counts
derived, link resolution, frontmatter contract, router integrity, version
equality across three manifests. Suite grew 18 → 71 tests over the drive;
every consistency defect from Pass 0 became a permanent regression guard.

## What the rounds caught that the author could no longer see

**Before the fan-out — the code-review lens.** A deep review of the
executable surface (prompt 3's shape: concrete failure scenarios only)
found three must-fix engine bugs, each with a one-shot repro: a
cross-session ledger whose "work compounded" classifier was satisfiable
without touching any prior work; a crash on the second breath over an empty
corpus; and an adversarial-refutation parser that counted a model echoing
its instructions as a survival vote. Its proposed tests went in verbatim;
the suite's growth to 71 tests began here.

**Round 1 — stranger + prose (parallel).** The stranger found a false
"covered" row in the repo's own precommitted oracle (a release marked tagged
that was never tagged), a private project name used twice inside a flagship
skill, and an undefined three-way notation (P1/P-1/M1) that "genuinely
confused me until I cross-read two other files." The prose auditor located
the accreted strata precisely: two files carrying an operator-log register
("the real Oracle, browser-background", a tooling diary, "differentiatingly
profound"), an honesty tic (34 hits of honest/honestly), and self-grading
("the strongest signal available"). Both reports converged on: good corpus,
thin internal-notes layer never scraped.

**Round 2 — cross-model refutation (different model family, high effort).**
16 findings. The real ones were deep: the MDL prose said "accepted only if
codelen decreases" while the code ratcheted on a ratio — and a two-item
corpus (`["alpha beta", "alpha beta"]`) got a "compression" accepted that
was *worse than raw encoding* (ratio 0.8). Fix: rules kept only if they pay,
yielding a provable never-worse-than-raw invariant, plus the ledger
classifier counting adjudication properly and the triangulation harness
refusing to manufacture the vague scalar its own skill forbids. Three
findings were **rejected with written reasons** (e.g. "do not publish until
empirical receipts exist" — the repo names those gaps rather than claiming
them); the rejections were committed as ledger entries, one explicitly
logged as a contested call.

**Round 3 — convergence.** One finding: a single leftover sentence still
describing the old absolute-codelen ratchet, 14 lines below the corrected
paragraph — exactly the class round 2 was scrubbing, in the same section.
Fixed; plus a half-sentence disclosure of the ratio-1.0 floor regime.

**The empty round — not measured.** Round 3's own verdict was "a fourth
adversarial round is not required," and the drive stopped there. By the
strict gate that is a convergence *verdict*, not a measured empty round —
the honest asterisk on this receipt, caught when this repo's own
stranger-test audited the claim. The doctrine stands corrected by its first
receipt: budget for the empty round; a reviewer predicting convergence is
not a round finding nothing.

## The shape of the yield

- Rounds found **34 accepted must-fix/should-fix defects** in an artifact
  that had already survived two frontier-model reviews.
- Every round found a defect class the *previous* round could not: leaks →
  formalism → residue-of-the-fix → nothing.
- The reviewer findings became the best test fixtures in the suite: the
  duplicate-corpus MDL counterexample and the same-session-churn ledger case
  are now permanent regression tests.
- The single most valuable find was the false "covered" row in the repo's
  own oracle file — the artifact's self-assessment failing its own
  discipline, caught only because the denominator was explicit enough to be
  audited row by row.

## End state

v0.2.0 tagged at the converged commit; release notes name what was reviewed
and the three remaining named gaps (no external-corpus compression curve,
no heterogeneous-panel receipts, a one-session ledger history); the
committed residue ledger carries six open items including the contested
publish-now-vs-wait call; the repo's skills installed into the operating
agent's own environment as the final self-application — and appeared live in
its skill list, the cheapest possible installability receipt.
